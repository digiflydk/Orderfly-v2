
# Brand Website: Schema â†’ UI Mapping

This document provides a one-to-one mapping between the Brand Website's Firestore schema fields, their corresponding UI elements in the Superadmin CMS, and the server actions used to save them.

## Introduction

This reference is intended for developers to ensure that the CMS UI stays in sync with the backend data model. Every field defined in the `website.config` and `website.designSystem` schemas should have a corresponding editable field in the UI.

---

## Core Config Mapping

**File:** `src/components/superadmin/brand-website/config/BrandWebsiteConfigForm.tsx`  
**Save Action:** `saveBrandWebsiteConfig(brandId, data)`

| Schema Path             | UI Section              | UI Field Label             | Input Type            |
| ----------------------- | ----------------------- | -------------------------- | --------------------- |
| `config.active`         | Config tab              | "Website Active"           | Switch                |
| `config.template`       | Config tab              | "Template"                 | Text input (disabled) |
| `config.domains[]`      | Config tab              | "Domains"                  | Dynamic list of inputs|
| `config.defaultLocationId` | Config tab           | "Default Location ID"      | Text input            |
