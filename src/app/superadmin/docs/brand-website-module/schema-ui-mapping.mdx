
# Brand Website: Schema â†’ UI Mapping

This document provides a one-to-one mapping between the Brand Website's Firestore schema fields and their corresponding UI elements in the Superadmin CMS. It clarifies which save action is responsible for persisting the data.

This mapping is based on the Zod schemas found in `src/lib/superadmin/brand-website/config-schemas.ts`.

## Core Config Mapping

**UI Location:** `/superadmin/brands/{brandId}/website/config` (General Tab)
**Save Action:** `saveBrandWebsiteConfig`

| Schema Path             | UI Label              | Input Type       | Notes                               |
| ----------------------- | --------------------- | ---------------- | ----------------------------------- |
| `active`                | "Website active"      | Toggle (Switch)  | Toggles the entire public website.  |
| `template`              | "Website template"    | Text (disabled)  | Currently locked to `template-1`.   |
| `domains[0]`            | "Primary domain"      | Text input       | The main domain for the website.    |
| `domains[1...]`         | "Additional domains"  | Repeating inputs | For domain aliases.                 |
| `defaultLocationId`     | "Default location"    | Select (Location)| Fallback for "Order Now" links.     |
